## Anti-Patterns Verdict

Does this look AI-generated? No, on the core surfaces.

**LLM assessment (A):** A Linear/Zed/Raycast-fluent user trusts the transcript view: hairline borders, one indigo accent used sparingly, thin scrollbars, honest `ctx/percent/cost` in a Zed-style status bar, a collapsed reasoning affordance, real window chrome. Not rounded-SaaS; no gradient heroes, tracked eyebrows, or glassmorphism. Where it slips is emptiness and inconsistency, not decoration: the home void and a settings rail that is 57% "Not available yet."

**Deterministic scan (B):** `detect.mjs --json` on `apps/desktop/src/components` and `.../settings` returned **0 findings, exit 0**, re-verified with per-file runs and `--no-config` (the clean result is genuine, not project-ignore suppression). No slop tells detected.

**Where they disagreed, resolved by measurement:**
- A flagged square-theme "radius drift" (`rounded-lg/xl/md` on cards) as a priority issue. Measured in the live app: `rounded-lg` -> `0px`, `rounded-xl` -> `0px`, `rounded-md` -> `0px`, literal clamps `rounded-[min(var(--radius-md),12px)]` -> `0px`. The token system binds these to `--radius: 0`, so they render square. The detector was right to stay silent. The only genuine drift is **bare `rounded` -> 4px** and `rounded-full` used on queued pills (round). Downgraded to Minor.
- A/B both flagged `muted-foreground` contrast for verification. Measured: **base `text-muted-foreground` passes AA** (4.6-6.2:1 across timestamp/labels/body). The failure is the **opacity variants**: `text-muted-foreground/60` on the "Add key" hint measured **2.88:1** (fails the 4.5 AA bar). `/70`, `/50`, `/40` are used on placeholders, secondary paths, and disabled rows and fall below AA. This is confirmed, not "verify."

**Visual overlays:** Not available. The Tauri webview needs `window.__TAURI__` to boot, so the standard live-server + detect.js injection flow does not apply; contrast/radius were measured directly in the running app instead.

## Overall Impression

The core loop, reading and steering a streaming agent run, is genuinely well made and on-brand: dense, honest, native, restrained. The app is dragged down at its edges. The single biggest opportunity is the first-run/home screen: for a product whose thesis is "the transcript is the product," the default landing screen has no transcript and no way to start one.

## What's Working

1. **Streaming telemetry + status bar** (ThreadView/ContextBar): per-panel `ctx/percent/cost` sliced to panel width, honest `--/--` fallbacks, truthful Working/Stopped/error states. Exactly the register's "honest state."
2. **The honest placeholder pattern** (`Placeholder` in panels.tsx): no fake toggles, a dashed empty state plus a concrete "Planned" list with real ticket refs. The pattern is right; only the quantity is the problem.
3. **The @ context picker + composer keyboard model** (Composer.tsx): atomic inline chips, Recent section, `@file:`/`@thread:` drill-in, mutually-exclusive slash vs mention handling, full Escape/Arrow/Enter/Tab semantics. Real Zed-parity craft on the primary surface.

## Priority Issues

**[P0] Home / empty state is a dead void with no primary action.**
- Why it matters: `HomePage.tsx` renders only a Sparkle + "What's up next?" heading; ~90% of the viewport is empty and the question promises an input the screen doesn't provide. It fails the first-run emotional moment and inverts "the transcript is the product" into "nothing is the product."
- Fix: put a real session-less composer on home (submit creates a thread), or a functional launcher (recent threads, Add project, quick model pick). Fill with utility, not filler.
- Suggested command: /impeccable onboard (then /impeccable layout)

**[P1] Providers "Search providers" input is clipped at the panel's right edge.**
- Why it matters: a visible rendering bug sits mid-way down the highest-trust screen (credentials), undercutting the exact trust the "keys never shown again" copy builds.
- Fix: constrain the API-keys header row to the panel content width; align the input to the card grid instead of bleeding past it. Audit the flex/width math in `ProvidersPanel`.
- Suggested command: /impeccable harden (then /impeccable polish)

**[P1] Opacity-reduced muted text fails WCAG AA (measured).**
- Why it matters: stated target is AA including placeholders at 4.5:1. Base `--muted-foreground` passes, but `text-muted-foreground/60` measured 2.88:1 on functional hint text ("Add key"), and `/70`, `/50`, `/40` (secondary paths, planned lists, disabled rows) fall below AA. Functional copy, not just decoration, is affected.
- Fix: stop using `/70`-and-below opacity on informational/interactive text. Introduce a dedicated secondary token at a verified AA lightness; reserve sub-AA strictly for decorative/disabled elements, and pair disabled state with more than dimming.
- Suggested command: /impeccable colorize (then /impeccable audit)

**[P2] Settings surface over-advertises: 8 of 14 categories are placeholders.**
- Why it matters: Appearance, Safety, Voice, Advanced, Gateway, Tools & Keys, MCP, Archived are all "Not available yet." Density-over-hand-holding is undercut when more than half the nav leads to empty rooms; it makes a small app feel padded.
- Fix: group placeholders under one "Coming soon" section or hide until backed; keep the rail to what's real (Model, Chat, Workspace, Memory, Providers, About).
- Suggested command: /impeccable distill

**[P2] Title-bar branch chip is hard-coded "main" (mocked git state).**
- Why it matters: the audience lives in git; a decorative-but-false status in the chrome is worse than no status, and this persona is the most likely to notice.
- Fix: wire a real `git -C <project.path>` branch/dirty read (already a tracked TODO), or hide the chip until it is real.
- Suggested command: /impeccable clarify (scope: honest status), backed by the real git command work

## Persona Red Flags

**Alex (power user):** No global command palette, no Cmd+N / Cmd+, ; every cross-screen action is mouse-only (gear in title bar, "+" in header, sidebar). The home screen gives him nowhere to type. Model/mode/thinking are reachable only from the composer's bottom-right cluster.

**Sam (a11y / keyboard / screen reader):** `text-muted-foreground/60` and below fail AA on functional text (measured 2.88:1). The @ picker and slash popups are custom `div`s with `onMouseDown preventDefault` and no `role="listbox"`/`aria-activedescendant`, so the highlighted option is not announced. Check the dev `animate-pulse` against `prefers-reduced-motion`.

**Riley (stress tester):** The Providers search input already overflows at default width; narrowing the resizable sidebar or the window will stress status-bar slices, chips, and menus. Raw `String(e)` from the updater can dump an unbounded string into a small card. The home void gets worse, not better, at large window sizes.

**The dogfooding Pi developer (project persona):** The branch chip lies ("main", mocked). The Debug get_state / raw JSON panel is genuinely useful and well-judged for this persona, but Advanced (raw RPC payloads / verbose logging) is a placeholder, so the person most likely to want diagnostics can't get them from the UI yet.

## Minor Observations

- Radius: `rounded-lg/md/xl` all compute to 0px (correct). Only bare `rounded` -> 4px and `rounded-full` on queued pills deviate; decide if pills are an intended exception alongside dots/switch-thumbs/avatars.
- Two "expand" metaphors share the Maximize2 icon for different scopes (composer editor vs full-screen panel).
- The @ picker root shows 4 disabled categories (`/40`) as inert rows every open; hide until implemented.
- Emerald status dot with a glow is a second accent hue in a one-accent brand.
- Settings modal backdrop lets the "What's up next?" home heading ghost through faintly behind the dialog (z-index/backdrop-opacity artifact).
- About host info is redundant (`platform: linux`, `OS type: linux`).

## Questions to Consider

- If the transcript is the product, why does the default landing screen contain no transcript and no way to start one? What if home *were* a composer?
- Is a settings rail that is 57% empty rooms more honest, or just louder about what is missing? Would six real categories feel more finished than fourteen mixed ones?
- For a keyboard-first audience, all cross-screen navigation is mouse-only. Is "native, not web-in-a-window" true without a command palette?
- The branch chip is mocked to always say "main." For a tool whose users live in git, is a decorative-but-false status worse than no status?
